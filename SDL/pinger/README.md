# Задание №2
## На основе кода разработанного в практическом занятии 1, разработать программу:
• Приложение сервис(pinger, heartbit) выполняющую функции отправки запроса версии БД Postgresql [16, 17], раз в 5 минут.
• Факт успешного или не успешное подключения к БД должен записываться в соответствующий поток вывода stdout / stderr.
• Если БД возвращает не типичный ответ на запрос версии, но не вызывает ошибку он должен записываться в поток вывода stdout.
• Приложение не должно зависать в случае сетевых проблем, ошибки подключения не должны отменять проверку через следующие 5 минут.
• Интервал между опросами (5 минут) должен задаваться через переменные среды окружения.
• Логин и пароль необходимый для подключения к БД теперь должны получаться через переменные среды окружения.
• Приложению через переменную среды окружения, дополнительно можно передать путь к файлу, куда будут дублироваться логи.
• Написать Dockerfile который упаковывает скомпилированное / интерпретируемое приложение и все необходимые для его работы зависимости, в том числе файл конфига.
• Написать Env файл для Docker содержащий работающие примеры значений переменных среды (пункты 1.5 — 1.7) для демонстрации.
• Собрать тегированный образ Docker на основе Dockerfile файла.
• Подключить собранный образ в Docker compose с postgresql docker образом, передав параметром путь до Env файла.
## Требования для сдачи приложения:
1. Расположить исходный код программы, Dockerfile, Env файл, Docker compose файл в отдельный публичный репозиторий или в отдельную ветку предыдущего репозитория.
2. Продемонстрировать работу приложения в случае успешного подключения к БД, в случае неуспешного
первого подключения к БД и успешного второго подключения к БД.
3. Ответить на вопросы преподавателя.
